Redis ​

相关概念 ​

Redis 是完全开源免费的，遵守BSD协议，是一个高性能的key-value数据库。

Redis 与其他 key - value 缓存产品相比：支持数据的持久化，多数据结构list， set, zset, hash等的存储支持数据备份。

特点 ​

  • 高性能，可持久化
  • key-value结构，支持多种数据类型
  • 支持事务，数据的原子性（要么不做/全做）

应用场景 ​

  • 缓存（读写性能优异）
  • 计数&消息系统（高并发、发布/订阅阻塞队列功能）
  • 分布式会话session&分布式锁（秒杀）

Redis vs Mongo ​

  • 存储方式不一样：key-value vs Document
  • 使用方式&可靠性不一样：MongoDB SQL & ACID支持
  • 应用场景不一样：高性能缓存 vS 海量数据分析

Redis安装 ​

在Ubuntu系统上安装Redis非常简单。以下是详细步骤：

1. 更新系统包索引 ​

首先，确保您的包索引是最新的：

bash
sudo apt update

2. 安装Redis ​

使用以下命令安装Redis：

bash
sudo apt install redis-server

3. 配置Redis ​

安装完成后，您可能需要配置Redis。默认配置文件位于 /etc/redis/redis.conf。

编辑Redis配置文件：

bash
sudo nano /etc/redis/redis.conf

一些常见的配置选项包括：

  • 更改绑定地址：默认情况下，Redis只监听localhost。要允许外部连接，您可以修改bind选项。

    plaintext
    bind 127.0.0.1 ::1

  • 启用持久化：Redis提供两种持久化方式，RDB和AOF。您可以在配置文件中启用并配置这些选项。

4. 启动和启用Redis ​

安装完成后，Redis服务会自动启动。您可以使用以下命令检查Redis服务状态：

bash
sudo systemctl status redis

您还可以使用以下命令启用Redis服务，使其在系统启动时自动启动：

bash
sudo systemctl enable redis

5. 测试Redis ​

使用Redis CLI客户端连接并测试Redis服务器：

bash
redis-cli

在Redis命令行中，您可以输入以下命令来检查Redis是否正常工作：

plaintext
ping

如果Redis正常工作，它会返回PONG。

6. 配置Redis安全（可选） ​

为了确保Redis服务器的安全性，建议您进行以下配置：

  • 设置密码：编辑/etc/redis/redis.conf，取消注释并设置requirepass选项：

    plaintext
    requirepass yourpassword

  • 限制访问：确保只有需要访问Redis的客户端才能连接。您可以通过防火墙或网络配置来实现这一点。

7. 允许远程访问（可选） ​

如果您需要从远程机器连接到Redis服务器，请修改/etc/redis/redis.conf文件中的bind地址和保护模式：

  • 将bind地址改为0.0.0.0：

    plaintext
    bind 0.0.0.0

  • 关闭保护模式（不建议在生产环境中关闭）：

    plaintext
    protected-mode no

重启Redis服务以应用更改：

bash
sudo systemctl restart redis

8. 使用Docker安装Redis（可选） ​

如果您更喜欢使用Docker，可以通过以下步骤来安装和运行Redis：

  1. 拉取Redis Docker镜像：

    bash
    sudo docker pull redis

  2. 运行Redis容器：

    bash
    sudo docker run --name redis-container -d redis

  3. 连接到Redis容器：

    bash
    sudo docker exec -it redis-container redis-cli

Redis 端口配置 ​

在Ubuntu中，Redis默认监听的端口是6379。如果您需要更改Redis的端口，可以通过修改Redis的配置文件来实现。以下是详细的步骤：

1. 编辑Redis配置文件 ​

Redis的配置文件通常位于 /etc/redis/redis.conf。您可以使用您喜欢的文本编辑器来编辑这个文件，例如 nano 或 vim：

bash
sudo nano /etc/redis/redis.conf

2. 找到并修改端口配置 ​

在配置文件中，找到以下行：

plaintext
port 6379

将 6379 更改为您希望Redis监听的端口号，例如 6380：

plaintext
port 6380

3. 保存并关闭配置文件 ​

如果您使用的是 nano，按 Ctrl + X，然后按 Y 确认保存，最后按 Enter 关闭文件。

4. 重启Redis服务 ​

为了使配置更改生效，您需要重启Redis服务：

bash
sudo systemctl restart redis

5. 验证Redis端口 ​

您可以使用以下命令来验证Redis是否在新的端口上监听：

bash
sudo netstat -tunlp | grep redis

或者使用 redis-cli 连接到新的端口：

bash
redis-cli -p 6380

6. 防火墙设置（可选） ​

如果您使用防火墙（例如 ufw），确保防火墙允许新的端口。假设您使用的是 ufw，可以使用以下命令允许新的端口：

bash
sudo ufw allow 6380
sudo ufw reload

完整步骤回顾 ​

  1. 打开Redis配置文件：

  2. 修改端口配置：

  3. 保存并关闭文件。

  4. 重启Redis服务：

  5. 验证Redis端口：

    bash

Nodejs中使用Redis ​

1.安装 ​

npm install -S redis

2.Nodejs链接Redis ​

在config中新建RedisConfig.js配置文件

javascript

import redis from "redis";

const options = {
    host: "192.168.1.112",
    port: 6379,
    detect_buffers: true,
    retry_strategy: function (options) {
        if (options.error && options.error.code === "ECONNREFUSED") {
            // End reconnecting on a specific error and flush all commands w
            //a individual error
            return new Error(" The server refused the connection");
        }
        if (options.total_retry_time > 1000 * 60 * 60) {
            // End reconnecting after a specific timeout and flush all commal
            // with a individual error
            return new Error("Retry time exhausted");
        }
        if (options.attempt > 10) {
            // End reconnecting with built in error
            return undefined;
        }
        // reconnect after
        return Math.min(options.attempt * 100, 3000);
    },
};

const client = redis.createClient(options);

// set数据
const setValue = (key, value) => {
    // 如果没有传值 直接返回
    if (typeof value === "undefined" || value === null || value === "") {
        return;
    }

    // 如果是字符串 直接set
    if (typeof value === "string") {
        return client.set(key, value);
    } else if (typeof value === "object") {
        // 如果是对象  set hset
        Object.keys(value).forEach((item) => {
            client.hset(key, item, value[item], redis.print);
        });
    }
};

const { promisify } = require("util");
const getAsync = promisify(client.get).bind(client);

// 获取set key的数据
const getValue = (key) => {
    return getAsync(key);
};

// 获取hset的数据
const getHValue = (key) => {
    return promisify(client.hgetall).bind(client)(key);
};

// const getValue = (key) => {
//     return client.get(key);
// };

export { client, setValue, getValue, getHValue };
